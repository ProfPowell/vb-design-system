Overview

The <icon-specimen> component presents a project's icon language: a curated, fixed set of icon names rendered across the sizing scale. Unlike <icon-set>, it doesn't search or fetch a manifest — it's a design specimen for showing which icons a project uses and how they read at each size, not a browser for finding icon names.

Attributes

Attribute Type Default Description
set string lucide Icon set directory under the icon base path.
names string — (required) Space/comma-separated list of curated icon names to display.
data-sizes string 1rem 1.5rem 2rem Space-separated sizing scale. Each icon renders once per size.

Rendering

Each cell renders via <icon-wc name="..." set="..."> — the same real icon-loading path (SVG fetch, caching, currentColor theming) used elsewhere in the system. Sizing comes from an inline font-size on each <icon-wc>, since the component sizes itself in em units relative to its own font-size.

Design system workflow

Use <icon-specimen> to document the small, curated set of icons a project actually uses — as opposed to <icon-set>, which browses the full catalog. A typical icon guidelines page shows:

  1. The curated icon list at every size the project uses (via data-sizes).
  2. Do/don't guidance authored alongside it (e.g. a data-list="do-dont" block) for usage rules such as stroke weight or minimum size.

Accessibility

  • Icon names are always visible as text (in a <code> row header), never conveyed by glyph alone.
  • The specimen renders as a native <table> with <th scope="row">/<th scope="col"> headers, so assistive tech can announce which icon and size a cell represents.
  • Icons render via <icon-wc>, which marks decorative glyphs aria-hidden by default.
