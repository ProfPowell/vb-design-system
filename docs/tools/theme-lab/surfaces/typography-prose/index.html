Prose & Inline Elements

Good typography starts with the paragraph. Line length, line height, and font size work together to produce text that feels natural to read. The ideal measure for body text sits somewhere between 55 and 75 characters per line — wide enough to feel spacious, narrow enough to keep the eye from losing its place.

Inline elements carry semantic weight. Use bold text to signal importance, and italic text for titles, technical terms, or gentle emphasis. Never use either purely for visual effect without meaning.

Code appears in a monospace face: const scale = tokens.size.xl;. It should stand out clearly from surrounding prose without disrupting the flow of reading.

Highlighted text draws attention to a specific phrase. It is useful for search results, warnings, or key takeaways that the reader should not miss.

Press Ctrl + Shift + I to open the browser developer tools. Keyboard indicators should feel tactile and distinct.

The CSS specification defines how browsers render visual presentation. The W3C maintains the standard.

Links connect ideas: read the type scale documentation or explore contrast ratio guidelines. Link styles should be distinguishable from surrounding text at a glance.

Small text is used for legal disclaimers, captions, and metadata. It should remain legible even at reduced size, and must meet contrast requirements.

Blockquote

Typography is the visual component of the written word. A reader notices bad typography immediately, even if they cannot name what is wrong. Good typography is invisible — it serves the content without drawing attention to itself.

— Matthew Butterick, Practical Typography

Blockquotes should be visually set apart from body text through indentation, a border accent, or a shift in weight — but never all three at once.

Lists

Unordered List

  • Choose a type scale with at least a 1.25 ratio between steps
  • Set body text to 16px or larger for comfortable reading
  • Keep line length between 55 and 75 characters
  • Use no more than two typefaces in a single interface
  • Ensure a minimum contrast ratio of 4.5:1 for body text

Ordered List

  1. Define your base font size in the :root
  2. Build a modular scale from that base
  3. Assign scale steps to semantic tokens (--text-sm, --text-lg, etc.)
  4. Apply tokens to components, never raw values
  5. Test across all viewports and at browser zoom levels

Description List

Measure
The width of a column of text, typically expressed in characters. Optimal measure for body copy is 45–75 characters per line.
Leading
The vertical space between lines of text, known in CSS as line-height. Body text typically requires a leading ratio of 1.4–1.6.
Kerning
The adjustment of space between specific pairs of characters to achieve visually consistent spacing. Not to be confused with tracking.
Tracking
Uniform letter-spacing applied across a word, phrase, or block of text. Often used with uppercase headings or display text.
Modular scale
A sequence of sizes derived by multiplying a base value by a fixed ratio. Common ratios include the perfect fourth (1.333), the golden ratio (1.618), and the major third (1.25).
